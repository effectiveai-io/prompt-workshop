# 문서 초안 평가 기준

10개 항목, 각 1~5점. 만점 50점.

## 채점 기준

- **1점**: 언급 없음 또는 완전히 일반적
- **2점**: 언급은 있지만 구체성 없음
- **3점**: 도메인 맥락이 일부 반영됨
- **4점**: 구체적이고 실행 가능한 수준
- **5점**: 전문가 수준의 완성도

## 평가 항목

### 1. 구조 완결성 (1~5)
- 1: 제목과 본문만 있음
- 2: 일부 섹션이 있지만 불완전
- 3: 요약, 배경, 본문, 결론 구조
- 4: 완전한 구조 + 목차로 전체 파악 가능
- 5: 구조 + 각 섹션의 길이/깊이가 적절하게 균형

### 2. 한 줄 요약 (1~5)
- 1: 요약 없음
- 2: 요약이 있지만 핵심이 불명확
- 3: 핵심 메시지가 담긴 요약
- 4: 읽지 않고 요약만 봐도 판단 가능한 수준
- 5: 요약 + 대상 독자 + 예상 소요 시간 명시

### 3. 배경/맥락 (1~5)
- 1: 배경 설명 없이 바로 본문
- 2: "이 문서의 목적은~" 수준
- 3: 왜 이 문서가 필요한지 설명
- 4: 배경 + 현재 상태 + 문제점 + 이 문서가 해결하려는 것
- 5: 배경 + 이해관계자 + 의사결정에 필요한 맥락까지 완비

### 4. 독자 수준 적합성 (1~5)
- 1: 대상 독자를 고려하지 않음
- 2: 일부 전문 용어를 설명
- 3: 대상 독자 수준에 맞는 용어 사용
- 4: 전문 용어 첫 등장 시 모두 설명 + 약어 풀어쓰기
- 5: 독자별 읽기 경로 제안 ("개발자는 3장부터, PM은 2장부터")

### 5. 근거와 수치 (1~5)
- 1: 주장만 있고 근거 없음
- 2: "~한 것으로 알려져 있다" 수준
- 3: 일부 근거나 사례 포함
- 4: 핵심 주장마다 수치/사례/출처 뒷받침
- 5: 수치 + 출처 + 수치의 한계/가정 조건까지 명시

### 6. 코드/기술 예시 (1~5)
- 1: 기술 문서인데 코드 예시 없음
- 2: 의사코드 수준
- 3: 실행 가능한 코드 스니펫 포함
- 4: 코드 + 언어 표시 + 설명 주석
- 5: 코드 + 실행 결과 + 에러 케이스 + "이렇게 하면 안 되는" 안티패턴

### 7. 시각적 설명 (1~5)
- 1: 텍스트만으로 나열
- 2: 표 1~2개 포함
- 3: 표 + 다이어그램 필요 표시
- 4: 표 + 다이어그램 + 흐름도로 복잡한 개념 시각화
- 5: 시각 자료 + 캡션 + 각 시각 자료가 설명하는 핵심 포인트 명시

### 8. 실행 가능한 결론 (1~5)
- 1: 결론 없이 끝남
- 2: 요약만 있고 다음 단계 없음
- 3: "다음에 이런 걸 해야 한다" 방향 제시
- 4: 누가 무엇을 언제까지 해야 하는지 액션 아이템
- 5: 액션 아이템 + 담당자 + 기한 + 성공 기준

### 9. 마크다운 활용 (1~5)
- 1: 서식 없는 텍스트 나열
- 2: 헤딩만 사용
- 3: 헤딩 + 리스트 + 코드 블록
- 4: 헤딩 + 리스트 + 코드 블록(언어 표시) + 표 + 인용
- 5: 전체 요소 활용 + Callout/Admonition + 접기/펼치기로 가독성 최적화

### 10. 유지보수 가능성 (1~5)
- 1: 일회성 문서 느낌
- 2: 날짜 표시 정도
- 3: 버전 + 최종 수정일 + 작성자
- 4: 변경 이력 + 관련 문서 링크 + 용어 정의
- 5: 변경 이력 + 링크 + 용어 + "이 문서가 outdated되는 조건" 명시

## 예상 점수대

| 프롬프트 수준 | 예상 점수 | 설명 |
|---|---|---|
| 빈 프롬프트 | 10~15점 | 기본적인 문서만 |
| 기본 구조 포함 | 18~24점 | 섹션 구조, 마크다운 활용 |
| 독자/맥락 포함 | 25~32점 | 배경, 근거, 독자 수준 반영 |
| 전문가 수준 | 33~40점 | 시각화, 액션 아이템, 유지보수 |
| 만점 50점 | 사실상 불가능 | 항상 개선 여지 있음 |
