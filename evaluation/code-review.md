# 코드 리뷰 평가 기준

10개 항목, 각 1~5점. 만점 50점.

**채점 원칙**: 각 점수의 기준은 "결과물에서 세거나 확인할 수 있는 것"입니다. AI의 주관적 판단이 아니라, 결과물에 해당 내용이 있는지 없는지로 채점합니다.

## 평가 항목

### 1. 네이밍 컨벤션 (1~5)
- 1점: 네이밍 관련 지적이 0건
- 2점: "네이밍을 확인하세요" 같은 일반 문구만 있음
- 3점: 코드에서 네이밍 위반을 1개 이상 구체적으로 지적 (변수명/메서드명 인용)
- 4점: 위반 지적 + 팀 규칙명을 명시 (예: "camelCase 규칙 위반")
- 5점: 위반 지적 + 팀 규칙 + 개선된 이름을 코드로 제안

### 2. 레이어 분리 (1~5)
- 1점: 코드 구조/레이어 관련 지적이 0건
- 2점: "구조를 개선하세요" 같은 일반 문구만 있음
- 3점: 레이어 위반을 1건 이상 지적하며 해당 코드 위치를 인용
- 4점: 위반 지적 + "이 로직은 X 레이어에 있어야 한다"고 이동 위치 명시
- 5점: 위반 지적 + 이동 위치 + 리팩토링된 코드 예시 포함

### 3. 보안 취약점 (1~5)
- 1점: 보안 관련 지적이 0건
- 2점: "보안에 주의하세요" 같은 일반 문구만 있음
- 3점: 코드에서 보안 위험을 1건 이상 지적하며 해당 라인/코드를 인용
- 4점: 위험 지적 + 공격 시나리오 설명 (예: "이 로그에서 계좌번호가 노출되면...")
- 5점: 위험 + 시나리오 + 수정 코드 예시 포함

### 4. 금액/데이터 정확성 (1~5)
- 1점: 데이터 타입/정밀도 관련 지적이 0건
- 2점: "정확성에 주의하세요" 같은 일반 문구만 있음
- 3점: 코드에서 정밀도 문제를 1건 이상 지적하며 해당 변수/연산을 인용
- 4점: 문제 지적 + 발생 가능한 오차 시나리오 설명 (예: "0.1 + 0.2 != 0.3 문제")
- 5점: 문제 + 시나리오 + Before/After 수정 코드 포함

### 5. 동시성/경쟁 조건 (1~5)
- 1점: 동시성 관련 지적이 0건
- 2점: "동시성을 고려하세요" 같은 일반 문구만 있음
- 3점: 동시 요청 시 위험한 코드 영역을 1건 이상 지적하며 해당 코드를 인용
- 4점: 위험 지적 + 구체적 시나리오 (예: "두 출금이 동시에 들어오면 잔액이...")
- 5점: 위험 + 시나리오 + 해결 코드 (락, 트랜잭션 격리 등) 포함

### 6. 에러 처리 (1~5)
- 1점: 에러 처리 관련 지적이 0건
- 2점: "예외 처리를 추가하세요" 같은 일반 문구만 있음
- 3점: 누락된 예외 케이스를 2건 이상 구체적으로 나열
- 4점: 케이스 나열 + 각 케이스의 영향 설명 (예: "NPE 발생 시 500 에러 반환")
- 5점: 케이스 + 영향 + 예외 처리 코드 예시 포함

### 7. 테스트 용이성 (1~5)
- 1점: 테스트 관련 지적이 0건
- 2점: "테스트를 작성하세요" 같은 일반 문구만 있음
- 3점: 테스트하기 어려운 코드를 1건 이상 지적하며 이유 설명
- 4점: 지적 + 테스트 가능한 구조로 변경하는 방향 제시 (예: "의존성을 주입하면...")
- 5점: 지적 + 방향 + 리팩토링 코드 + 테스트 코드 예시 포함

### 8. 리뷰 심각도 분류 (1~5)
- 1점: 모든 지적이 동일한 수준으로 나열 (구분 없음)
- 2점: 일부 지적에만 "중요" 또는 "참고" 같은 표시가 있음
- 3점: 전체 지적에 3단계 분류 사용 (예: MUST/SHOULD/NICE)
- 4점: 3단계 분류 + 각 지적에 분류 이유가 1줄 이상 포함
- 5점: 분류 + 이유 + "이 순서로 수정하세요" 우선순위 제안

### 9. 코드 예시 품질 (1~5)
- 1점: 리뷰 전체에 코드 블록이 0개
- 2점: 코드 블록이 있지만 의사코드 수준 (실행 불가)
- 3점: 실행 가능한 코드 예시가 1개 이상 포함
- 4점: 지적 사항의 50% 이상에 Before/After 코드 쌍이 포함
- 5점: 주요 지적 전부에 Before/After 코드 + "왜 이렇게 바꾸는지" 설명

### 10. 도메인 우선순위 (1~5)
- 1점: 리뷰 항목이 발견 순서대로 나열 (우선순위 없음)
- 2점: 대략적 순서가 있지만 기준이 명시되지 않음
- 3점: "보안 이슈부터 봤습니다" 같은 우선순위 기준 1줄 이상 명시
- 4점: 우선순위 기준 명시 + 실제로 그 순서대로 리뷰가 구성됨
- 5점: 기준 + 순서 + "이 서비스에서 X가 Y보다 중요한 이유" 근거 포함

## 예상 점수대

| 프롬프트 수준 | 예상 점수 | 설명 |
|---|---|---|
| 빈 프롬프트 ("코드 리뷰해줘") | 12~18점 | 일반적 리뷰, 코드 인용 없음 |
| 기본 컨벤션 포함 | 20~26점 | 네이밍·구조 지적 시작 |
| 도메인 맥락 포함 | 27~34점 | 보안·금액·동시성 지적 등장 |
| 코드 예시 요구 포함 | 34~40점 | Before/After 코드 포함 |
| 전문가 수준 프롬프트 | 40~45점 | 시나리오+코드+우선순위 |
| 만점 50점 | 사실상 불가능 | 모든 항목 5점은 매우 어려움 |
