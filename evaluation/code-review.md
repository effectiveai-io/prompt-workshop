# 코드 리뷰 평가 기준

각 항목 0~2점. 0: 미반영, 1: 부분 반영, 2: 완전 반영. 만점 10점.

## 평가 항목

### 1. 팀 컨벤션 반영 (0~2점)
- 0점: 일반적인 코딩 규칙만 언급
- 1점: 일부 팀 규칙이 반영됨
- 2점: 지침서에 명시된 팀 컨벤션이 정확히 적용됨

### 2. 보안/성능 관점 (0~2점)
- 0점: 보안/성능 언급 없음
- 1점: 일반적인 보안/성능 주의사항만 언급
- 2점: 코드에 맞는 구체적인 보안/성능 이슈를 지적

### 3. 비즈니스 맥락 이해 (0~2점)
- 0점: 코드 문법/구조만 리뷰
- 1점: 일부 비즈니스 로직을 언급
- 2점: 도메인 특화 리뷰 (금액 정확성, 규제 준수 등)

### 4. 구체적 개선 제안 (0~2점)
- 0점: "수정하세요" 수준의 추상적 피드백
- 1점: 방향은 맞지만 구체적 코드가 없음
- 2점: Before/After 코드 예시가 포함된 제안

### 5. 심각도 분류 (0~2점)
- 0점: 모든 지적이 같은 수준
- 1점: 일부 항목에만 심각도 표시
- 2점: 모든 지적에 [MUST]/[SHOULD]/[NICE] 등 심각도가 명확히 분류됨
