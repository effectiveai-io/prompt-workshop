# 코드 리뷰 평가 기준

10개 항목, 각 1~5점. 만점 50점.
autoresearch의 val_bpb처럼, 만점에 도달하는 것은 사실상 불가능합니다.
사이클을 돌수록 점수가 조금씩 올라가는 것이 목표입니다.

## 채점 기준

각 항목을 1~5점으로 채점합니다:
- **1점**: 언급 없음 또는 완전히 일반적
- **2점**: 언급은 있지만 구체성 없음
- **3점**: 팀/도메인 맥락이 일부 반영됨
- **4점**: 구체적이고 실행 가능한 수준
- **5점**: 코드 예시 + 근거 + 대안까지 포함된 전문가 수준

## 평가 항목

### 1. 네이밍 컨벤션 (1~5)
- 1: 네이밍 관련 언급 없음
- 2: "네이밍을 확인하세요" 수준
- 3: 팀 규칙 일부 적용 (예: camelCase 언급)
- 4: 팀 규칙을 정확히 적용하여 위반 지적
- 5: 위반 지적 + 개선된 이름 제안 + 네이밍 근거 설명

### 2. 레이어 분리 (1~5)
- 1: 코드 구조 관련 언급 없음
- 2: "구조를 개선하세요" 수준
- 3: 레이어 역할을 일부 언급 (예: "컨트롤러에 로직이 많다")
- 4: 팀의 레이어 규칙을 기준으로 구체적 위반 지적
- 5: 위반 지적 + 리팩토링 코드 예시 제시

### 3. 보안 취약점 (1~5)
- 1: 보안 관련 언급 없음
- 2: "보안에 주의하세요" 수준의 일반론
- 3: 도메인 관련 보안 이슈 언급 (예: "민감정보 주의")
- 4: 코드에서 구체적인 보안 위험 지적 (예: "이 줄에서 계좌번호가 로그에 노출")
- 5: 위험 지적 + 수정 코드 + 보안 가이드라인 참조

### 4. 금액/데이터 정확성 (1~5)
- 1: 데이터 타입/정확성 관련 언급 없음
- 2: "정확성에 주의하세요" 수준
- 3: 금액 관련 타입 언급 (예: "BigDecimal을 고려하세요")
- 4: 코드에서 구체적 위험 지적 (예: "double로 금액 계산 시 부동소수점 오류 가능")
- 5: 위험 지적 + Before/After 코드 + 엣지 케이스 시나리오

### 5. 동시성/경쟁 조건 (1~5)
- 1: 동시성 관련 언급 없음
- 2: "동시성을 고려하세요" 수준
- 3: 잠재적 동시성 이슈 영역 언급
- 4: 구체적 경쟁 조건 시나리오 설명 (예: "두 출금 요청이 동시에 들어오면")
- 5: 시나리오 + 해결 코드 (락, 트랜잭션 등) + 테스트 방법 제안

### 6. 에러 처리 (1~5)
- 1: 에러 처리 관련 언급 없음
- 2: "예외 처리를 추가하세요" 수준
- 3: 누락된 예외 케이스 1~2개 지적
- 4: 체계적으로 누락된 케이스 나열 + 에러 메시지 개선 제안
- 5: 케이스 나열 + 커스텀 예외 클래스 + 에러 응답 코드 예시

### 7. 테스트 용이성 (1~5)
- 1: 테스트 관련 언급 없음
- 2: "테스트를 작성하세요" 수준
- 3: 테스트하기 어려운 부분을 지적
- 4: 테스트 가능한 구조로 리팩토링 방향 제안
- 5: 리팩토링 코드 + 테스트 케이스 예시 포함

### 8. 리뷰 심각도 분류 (1~5)
- 1: 모든 지적이 같은 수준으로 나열
- 2: 일부 항목에 "중요" 표시
- 3: 3단계 구분 사용 ([MUST]/[SHOULD]/[NICE] 등)
- 4: 일관된 분류 + 각 수준의 기준이 명확
- 5: 분류 + 우선순위 근거 + "이것부터 먼저 고치세요" 순서 제안

### 9. 코드 예시 품질 (1~5)
- 1: 개선 제안에 코드가 전혀 없음
- 2: 의사코드 수준의 방향만 제시
- 3: 일부 지적에 코드 스니펫 포함
- 4: 대부분의 지적에 Before/After 코드 포함
- 5: 모든 주요 지적에 Before/After + 왜 이렇게 바꿔야 하는지 설명

### 10. 도메인 우선순위 (1~5)
- 1: 리뷰 항목에 우선순위 없이 나열
- 2: 일반적인 순서로 나열 (문법 → 로직 → 성능)
- 3: 도메인 특성을 반영한 순서 (예: "보안 이슈를 먼저 봤습니다")
- 4: 도메인별 우선순위를 명시하고 그 순서로 리뷰
- 5: 우선순위 + 근거 + "이 서비스에서는 X가 Y보다 중요한 이유" 설명

## 예상 점수대

| 프롬프트 수준 | 예상 점수 | 설명 |
|---|---|---|
| 빈 프롬프트 ("코드 리뷰해줘") | 10~15점 | 일반적인 리뷰만 |
| 기본 컨벤션 포함 | 18~24점 | 네이밍, 구조 반영 |
| 도메인 맥락 포함 | 25~32점 | 보안, 금액, 동시성 반영 |
| 코드 예시 + 우선순위 포함 | 33~40점 | 전문가 수준 접근 |
| 완벽에 가까운 프롬프트 | 41~45점 | 매우 어려움 |
| 만점 50점 | 사실상 불가능 | 항상 개선 여지 있음 |
