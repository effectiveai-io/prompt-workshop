# TC 설계 평가 기준

10개 항목, 각 1~5점. 만점 50점.

**채점 원칙**: 각 점수의 기준은 "결과물에서 세거나 확인할 수 있는 것"입니다. AI의 주관적 판단이 아니라, 결과물에 해당 내용이 있는지 없는지로 채점합니다.

## 평가 항목

### 1. 정상 시나리오 (1~5)
- 1점: 정상 시나리오 TC가 0건
- 2점: "정상 동작 확인" 같은 모호한 TC가 1건
- 3점: 구체적 입력값이 포함된 정상 TC가 2건 이상
- 4점: 정상 TC 3건 이상 + 각 TC에 서로 다른 입력 조합
- 5점: 다양한 입력 조합 + 각 단계의 상태 변화(DB, 응답 등)까지 검증

### 2. 비정상 시나리오 (1~5)
- 1점: 비정상/에러 TC가 0건
- 2점: "에러 케이스 확인" 같은 모호한 TC가 1건
- 3점: 구체적 비정상 TC가 2건 이상 (예: 잘못된 입력, 권한 없음)
- 4점: 비정상 TC 3건 이상 + 서로 다른 실패 유형 (입력/권한/타임아웃 등)
- 5점: 다양한 실패 유형 + 복합 실패 시나리오 (예: 동시 요청 + 잔액 부족)

### 3. 경계값 테스트 (1~5)
- 1점: 경계값 TC가 0건
- 2점: "최대값 확인" 같은 모호한 TC만 있음
- 3점: 구체적 경계값 TC가 2건 이상 (예: 0원, 최대 금액)
- 4점: 경계값 + 경계 직전/직후 TC 포함 (예: 한도 999, 1000, 1001)
- 5점: 경계값 + 직전/직후 + 오버플로우/언더플로우 시나리오

### 4. 우선순위 분류 (1~5)
- 1점: TC에 우선순위 표시가 0건
- 2점: 일부 TC에만 "중요" 표시 (전체의 30% 미만)
- 3점: 전체 TC에 P1/P2/P3 분류가 적용됨
- 4점: 분류 + 각 TC에 "왜 이 우선순위인지" 이유가 1줄 이상
- 5점: 분류 + 이유 + 도메인 근거 (예: "금액 관련이므로 P1")

### 5. 사전 조건 (1~5)
- 1점: TC에 사전 조건이 0건
- 2점: "로그인 상태" 같은 1줄 사전 조건만 있음
- 3점: 사용자 상태 + 데이터 상태가 모두 명시된 TC가 50% 이상
- 4점: 사용자/데이터/시스템 상태 모두 명시 + 설정 방법 힌트
- 5점: 모든 상태 명시 + SQL/API 수준의 구체적 설정 방법 포함

### 6. 테스트 단계 구체성 (1~5)
- 1점: "기능을 테스트한다" 수준의 TC가 50% 이상
- 2점: 번호 매긴 단계가 있지만 입력값이 없는 TC가 50% 이상
- 3점: 번호 + 구체적 입력값이 포함된 TC가 50% 이상
- 4점: 번호 + 입력값 + 각 단계의 기대 반응이 포함된 TC가 50% 이상
- 5점: 번호 + 입력값 + 기대 반응 + 확인할 화면/API 경로까지 포함

### 7. 기대 결과 명확성 (1~5)
- 1점: "정상 동작해야 함" 수준의 기대 결과가 50% 이상
- 2점: 기대 결과가 있지만 구체적 값이 없는 TC가 50% 이상
- 3점: 구체적 기대값 (HTTP 코드, 메시지 등)이 포함된 TC가 50% 이상
- 4점: 기대값 + 검증 방법 (UI 확인, DB 쿼리 등)이 포함된 TC가 50% 이상
- 5점: 기대값 + 검증 방법 + 실패 시 예상 증상까지 명시

### 8. 보안/규제 시나리오 (1~5)
- 1점: 보안/규제 관련 TC가 0건
- 2점: "보안 확인" 같은 모호한 TC가 1건
- 3점: 인증/인가 TC가 2건 이상 (예: 미로그인, 타인 데이터 접근)
- 4점: 인증/인가 + 도메인 규제 TC (출금 한도, 본인 인증 등) 포함
- 5점: 규제 TC + 우회 시나리오 (예: 한도 분할 회피) + 감사 로그 검증

### 9. 데이터 무결성 (1~5)
- 1점: 데이터 검증 TC가 0건
- 2점: "데이터 확인" 같은 모호한 TC가 1건
- 3점: CRUD 후 데이터 상태 확인 TC가 2건 이상
- 4점: 데이터 확인 + 트랜잭션 롤백/부분 실패 시 데이터 상태 TC 포함
- 5점: 롤백 + 동시성 데이터 충돌 + 복구 시나리오 TC 포함

### 10. 형식 일관성 (1~5)
- 1점: TC마다 포함 항목이 다름 (형식 불일치)
- 2점: 대략적 구조는 있지만 누락 필드가 30% 이상
- 3점: TC ID, 우선순위, 단계가 전체 TC의 80% 이상에 포함
- 4점: 모든 TC가 동일한 필드 구성 (ID, 우선순위, 사전조건, 단계, 기대결과)
- 5점: 동일 필드 + 자동화 프레임워크에 연동 가능한 형식 (태그, 카테고리 등)

## 예상 점수대

| 프롬프트 수준 | 예상 점수 | 설명 |
|---|---|---|
| 빈 프롬프트 | 12~18점 | 기본 TC, 구체성 없음 |
| 기본 형식 포함 | 20~26점 | 구조화된 TC, 일부 경계값 |
| 도메인 맥락 포함 | 27~34점 | 보안/규제/데이터 TC |
| 전문가 수준 | 34~42점 | 복합 시나리오, 검증 방법 |
| 만점 50점 | 사실상 불가능 | 모든 항목 5점은 매우 어려움 |
