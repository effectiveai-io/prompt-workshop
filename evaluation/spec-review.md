# 기획서 리뷰 평가 기준

각 항목 0~2점. 0: 미반영, 1: 부분 반영, 2: 완전 반영. 만점 10점.

## 평가 항목

### 1. 예외 케이스 발견 (0~2점)
- 0점: 기획서에 적힌 내용만 확인
- 1점: 1~2개의 누락된 케이스를 지적
- 2점: 체계적으로 예외 케이스를 도출 (입력 오류, 동시성, 권한 등)

### 2. 기술 실현 가능성 (0~2점)
- 0점: 기술적 관점 없이 기획만 리뷰
- 1점: "구현이 어려울 수 있다" 수준의 일반적 언급
- 2점: 현재 아키텍처 기반으로 구체적인 기술 제약/대안을 제시

### 3. 모호한 표현 지적 (0~2점)
- 0점: 모호한 표현을 그대로 통과
- 1점: 일부 모호한 표현을 지적
- 2점: "적절히", "필요시" 같은 모호한 표현을 모두 찾아내고 구체화 요청

### 4. 기존 정책과의 정합성 (0~2점)
- 0점: 새 기획만 독립적으로 리뷰
- 1점: 일부 기존 기능과의 관계를 언급
- 2점: 기존 정책/기능과의 충돌 가능성을 구체적으로 지적

### 5. 질문 형태의 피드백 (0~2점)
- 0점: "~해야 합니다" 식의 일방적 지적
- 1점: 일부 질문이 포함되지만 대부분 지시형
- 2점: "~한 경우는 어떻게 처리하나요?" 식의 질문 형태로 피드백
