# 기획서 리뷰 평가 기준

10개 항목, 각 1~5점. 만점 50점.

**채점 원칙**: 각 점수의 기준은 "결과물에서 세거나 확인할 수 있는 것"입니다. AI의 주관적 판단이 아니라, 결과물에 해당 내용이 있는지 없는지로 채점합니다.

## 평가 항목

### 1. 예외 케이스 발견 (1~5)
- 1점: 기획서 내용 요약만 있고 예외 케이스 지적이 0건
- 2점: "예외 상황을 고려하세요" 같은 일반 문구만 있음
- 3점: 구체적 예외 케이스를 2건 이상 지적 (시나리오 포함)
- 4점: 예외 3건 이상 + 유형별 분류 (입력오류/동시성/권한/타이밍 등)
- 5점: 유형별 분류 + 각 예외의 발생 빈도/영향도 분석 포함

### 2. 기술 실현 가능성 (1~5)
- 1점: 기술 관점 언급이 0건
- 2점: "구현이 어려울 수 있다" 같은 일반 문구만 있음
- 3점: 구체적 기술 제약을 1건 이상 지적 (예: "실시간 가격 조회 API 호출 빈도")
- 4점: 제약 지적 + 대안 제시 (예: "폴링 대신 웹소켓으로...")
- 5점: 제약 + 대안 + 예상 공수/의존성 분석 포함

### 3. 모호한 표현 지적 (1~5)
- 1점: 모호한 표현("적절히", "필요시" 등) 지적이 0건
- 2점: "좀 더 구체적으로" 같은 일반 피드백만 있음
- 3점: 모호한 표현을 2건 이상 인용하며 지적
- 4점: 모호 표현 인용 + 각각에 대해 "이런 뜻인가요?" 확인 질문
- 5점: 인용 + 질문 + 구체화 예시 제안 (예: "적절히" → "3초 이내")

### 4. 기존 정책 정합성 (1~5)
- 1점: 기존 기능/정책과의 관계 언급이 0건
- 2점: "기존 기능과 충돌이 없는지 확인하세요" 같은 일반 문구
- 3점: 관련 기존 기능을 1건 이상 구체적으로 언급
- 4점: 기존 기능 언급 + 충돌 가능성 시나리오 설명
- 5점: 충돌 시나리오 + 해결 방안 + 영향 범위 분석

### 5. 사용자 시나리오 (1~5)
- 1점: 사용자 관점 언급이 0건
- 2점: "사용자 경험을 고려하세요" 같은 일반 문구
- 3점: 구체적 사용자 시나리오를 1건 이상 작성 (행동 순서 포함)
- 4점: 사용자 유형 2종 이상 구분하여 각각의 시나리오 작성
- 5점: 유형별 시나리오 + 이탈/혼란 발생 가능 지점 지적

### 6. 데이터 모델 검토 (1~5)
- 1점: 데이터 관련 언급이 0건
- 2점: "데이터 구조를 정의하세요" 같은 일반 문구
- 3점: 필요한 데이터 항목을 2개 이상 구체적으로 언급
- 4점: 데이터 항목 + 데이터 흐름(입력→저장→조회) 설명
- 5점: 흐름 + 마이그레이션 필요성/인덱싱/확장성 검토 포함

### 7. 비기능 요구사항 (1~5)
- 1점: 비기능 요구사항(성능, 가용성 등) 언급이 0건
- 2점: "성능을 고려하세요" 같은 일반 문구
- 3점: 비기능 항목을 1건 이상 구체적으로 언급 (예: "응답 시간 3초 이내")
- 4점: 비기능 항목 2건 이상 + 각각에 수치 기준 제안
- 5점: 수치 기준 + 측정 방법 + 미달 시 대응 방안

### 8. 일정/리소스 타당성 (1~5)
- 1점: 일정/리소스 관련 언급이 0건
- 2점: "일정이 빠듯해 보인다" 같은 일반 문구
- 3점: 일정 위험을 1건 이상 구체적으로 지적
- 4점: 위험 지적 + 기능별 공수 예측 또는 단계별 릴리스 제안
- 5점: 공수 예측 + 리스크 + 대안 일정 + 우선순위별 릴리스 제안

### 9. 질문 형태 피드백 (1~5)
- 1점: 모든 피드백이 "~해야 합니다" 지시형
- 2점: 질문이 전체 피드백의 20% 미만
- 3점: 질문이 전체 피드백의 50% 이상
- 4점: 질문 + 각 질문에 맥락 설명이 포함 (예: "~한 경우가 있을 수 있어서 묻습니다")
- 5점: 질문 + 맥락 + 기획자가 선택할 수 있는 선택지 2개 이상 제시

### 10. 리뷰 구조화 (1~5)
- 1점: 지적 사항이 순서 없이 나열
- 2점: 대략적 묶음은 있지만 라벨/분류 없음
- 3점: 영역별 분류 태그 사용 (예: [기능] [기술] [UX])
- 4점: 분류 + 심각도 표시 + 요약 테이블 포함
- 5점: 분류 + 심각도 + 요약 + "이것만 해결하면 출시 가능" 판단 포함

## 예상 점수대

| 프롬프트 수준 | 예상 점수 | 설명 |
|---|---|---|
| 빈 프롬프트 | 12~18점 | 표면적 리뷰 |
| 기본 기준 포함 | 20~26점 | 예외, 모호 표현 지적 |
| 도메인 맥락 포함 | 27~34점 | 기술, 정책, 데이터 |
| 전문가 수준 | 34~42점 | 비기능, 일정, 구조화 |
| 만점 50점 | 사실상 불가능 | 모든 항목 5점은 매우 어려움 |
